#<a name="media-services-playready-license-template-overview"></a>媒體服務 PlayReady 授權範本概觀

Azure 媒體服務現在會提供可將 Microsoft PlayReady 授權的服務。 當使用者播放程式 (例如，Silverlight) 嘗試播放您 PlayReady 受保護的內容時，會傳送要求給授權服務取得授權。 如果授權服務核准要求，它會發出授權會傳送給客戶，並可用於解密並播放指定的內容。

媒體服務也會提供可讓您設定 PlayReady 授權的 Api。 授權包含的權限及您要 PlayReady DRM 執行階段強制當使用者嘗試播放受保護的內容時的限制。
以下是您可以指定的 PlayReady 授權限制的部分範例︰

- 從授權是有效的日期時間。
- 授權到期時的日期時間值。 
- 如要儲存在持續存放在用戶端的授權。 常設授權通常用來允許內容的離線播放。
- 最小的安全性層級的播放程式必須以播放您的內容。 
- Audio\video 內容的輸出控制項輸出保護層級。 
- 如需詳細資訊，請參閱[PlayReady 法規遵循的規則](https://www.microsoft.com/playready/licensing/compliance/)文件中的 [輸出控制項] 區段 (3.5)。

>[AZURE.NOTE]目前，您只能在設定 （此權限是必要） 的 PlayReady 授權 PlayRight。 PlayRight 可讓用戶端來播放內容。 PlayRight 也可讓設定限制特定播放。 如需詳細資訊，請參閱[PlayReadyPlayRight](media-services-playready-license-template-overview.md#PlayReadyPlayRight)。

若要設定 PlayReady 授權使用媒體服務，您必須設定媒體服務 PlayReady 授權範本。 XML 中定義的範本。

下列範例顯示最簡單的 （和最常見的） 範本，以設定基本串流授權。 使用此授權，您的用戶端能夠播放您 PlayReady 受保護的內容。
    
    <?xml version="1.0" encoding="utf-8"?>
    <PlayReadyLicenseResponseTemplate xmlns:i="http://www.w3.org/2001/XMLSchema-instance" 
                                      xmlns="http://schemas.microsoft.com/Azure/MediaServices/KeyDelivery/PlayReadyTemplate/v1">
      <LicenseTemplates>
        <PlayReadyLicenseTemplate>
          <ContentKey i:type="ContentEncryptionKeyFromHeader" />
          <PlayRight />
        </PlayReadyLicenseTemplate>
      </LicenseTemplates>
    </PlayReadyLicenseResponseTemplate>

XML 符合 PlayReady 授權範本 XML 結構描述定義 PlayReady 授權範本 XML 結構描述] 區段中。

媒體服務也會定義一組可以用於序列化和還原序列化與 XML 的.NET 類別。 主要類別的說明，請參閱[媒體服務.NET 類別](media-services-playready-license-template-overview.md#classes)。 用來設定授權範本。

如需使用.NET 類別，以設定 PlayReady 授權範本的端對端範例，請參閱[使用 PlayReady 動態加密和授權服務](https://msdn.microsoft.com/library/azure/dn783467.aspx)。

##<a id="classes"></a>若要設定授權範本所使用的媒體服務.NET 類別

以下是在主要的.NET 類別用來設定媒體服務 PlayReady 授權範本。 這些類別對應至[PlayReady 授權範本 XML 結構描述](media-services-playready-license-template-overview.md#schema)中定義的類型。

[MediaServicesLicenseTemplateSerializer](https://msdn.microsoft.com/library/azure/microsoft.windowsazure.mediaservices.client.contentkeyauthorization.mediaserviceslicensetemplateserializer.aspx)類別用來序列化和還原序列化與媒體服務授權範本 XML。

###<a name="playreadylicenseresponsetemplate"></a>PlayReadyLicenseResponseTemplate

[PlayReadyLicenseResponseTemplate](https://msdn.microsoft.com/library/azure/microsoft.windowsazure.mediaservices.client.contentkeyauthorization.playreadylicenseresponsetemplate.aspx)此課程代表回應傳回給使用者的範本。 包含自訂資料字串之間的授權伺服器和應用程式 （可能是很適合用於自訂應用程式邏輯），以及一或多個授權範本清單的欄位。

這是在範本階層中的 「 上方層級 」 類別。 表示回應範本包含授權範本的清單，並授權範本包含 （直接或間接） 的所有序列化構成範本資料的類別。


###<a name="playreadylicensetemplate"></a>PlayReadyLicenseTemplate

[PlayReadyLicenseTemplate](https://msdn.microsoft.com/library/azure/microsoft.windowsazure.mediaservices.client.contentkeyauthorization.playreadylicensetemplate.aspx)類別均代表建立 PlayReady 授權傳回給使用者的授權範本。 包含上授權及任何權限或限制時使用的內容金鑰 PlayReady DRM 執行階段強制執行的內容索引鍵的資料。


###<a id="PlayReadyPlayRight"></a>PlayReadyPlayRight

[PlayReadyPlayRight](https://msdn.microsoft.com/library/azure/microsoft.windowsazure.mediaservices.client.contentkeyauthorization.playreadyplayright.aspx)此課程代表 PlayReady 授權的 PlayRight。 授與使用者能夠播放限制零或更多設定授權及 PlayRight 本身 （用於播放特定原則） 上的內容。 大部分的 PlayRight 的原則有處理控制的輸出可透過播放內容類型的輸出限制和使用指定的輸出時必須放在任何限制。 例如，如果 DigitalVideoOnlyContentRestriction 已啟用，然後 DRM 執行階段將只允許視訊會顯示在上方數位輸出 （類比視訊輸出不允許將內容）。

>[AZURE.IMPORTANT]這種限制可以很強大，但也可能會影響消費者體驗。 如果輸出保護設定太多限制，請內容可能無法在某些用戶端上播放。 如需詳細資訊，請參閱[PlayReady 法規遵循的規則](https://www.microsoft.com/playready/licensing/compliance/)文件。

如需哪些保護層級的支援 Silverlight 的範例，請參閱︰[輸出保護支援 Silverlight](http://go.microsoft.com/fwlink/?LinkId=617318)。
